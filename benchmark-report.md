## まとめ

### 速度比較（1000ノート時の全文検索 avg）

```
SQLite (local):  0.47ms  ████
Hybrid:          0.57ms  █████
Turso (cloud):  ~38ms    ████████████████████████████████████████  (~80x slower)
```

### 結論

| 観点 | 結論 |
|------|------|
| **読み取り速度** | SQLite ≈ Hybrid >> Turso（約80倍の差） |
| **書き込み速度** | SQLite ≈ Hybrid >> Turso（約90倍の差） |
| **Hybrid の優位性** | ローカル速度を維持しつつクラウド同期でバックアップ・複数端末共有 |
| **Turso 単体の用途** | サーバーサイド（API サーバー等）からの直接アクセス用 |
| **ボトルネック** | Turso のレイテンシはネットワーク往復（~30ms/query）が支配的 |
| **スケーラビリティ** | SQLite/Hybrid は 1000件でもサブミリ秒。10,000件以上の測定は今後の課題 |

### 推奨構成

- **CLI / デスクトップ用途** → `hybrid` モード一択（ローカル速度 + クラウドバックアップ）
- **サーバー用途（API経由）** → `turso` 単体（ファイルシステム不要）
- **オフライン利用** → `sqlite` 単体
